# Jouda Context Network

## Location

The context network for this project is located at: `./context/`

## Purpose

This network keeps persistent project knowledge that should survive across agent sessions:

- Current project state and active work
- Architectural and security decisions
- Domain relationships between JoudaApp, Inventory, Android, and Edge Functions
- Operational workflows for deployment, sync, and release

## Source of Truth

`AGENTS.md` is the top-level system reference. The context network is the navigable working memory built around it.

When facts conflict:

1. Check the code and live Supabase state where possible.
2. Update `AGENTS.md` for system-level facts.
3. Update the relevant `context/` node for relationships, rationale, and operational memory.

## What Belongs Here

| Belongs in `context/` | Belongs elsewhere |
|---|---|
| Current state, rationale, relationships, risks | Source code |
| Cross-domain impact notes | Build artifacts |
| Agent navigation and maintenance notes | Long audit reports |
| Operational checklists | User-facing README content |

Keep files short. Add detail only when it helps future agents avoid re-discovery.
